Creating Decals with Easy Decal

Easy Decal takes any image or images able to be read by Java and converts them into custom decals or "sprays".

Decals are stored in WAD files
The WADs generated by Easy Decal will be placed in the same directory as the source images, with the same name as the source images except with a .WAD extension.

There are two ways to create decals: either by drag-and-drop, or using the command line.

To find out how to install a decal once it's been created, see Installing Decals.

Drag 'n' Drop

Use the following instructions to create decals on Windows using drag 'n' drop.

Note: due to some restriction in drag-droping files onto Windows batch scripts, you will only be able to create up to 20 (twenty) decals at a time this way. Attempt any more and Windows will show you an error message.

1. Open the Easy Decal Folder
Open My Computer and browse to the easy-decal folder under C:\Program Files.
2. Locate Your Source Images
Open another My Computer or Windows Explorer window, and browse to the directory where your images are.
You can also drag and drop images from a Search window.
3. Drag the Images onto EasyDecal_dragNdrop.bat
Select all the images you want to create decals for, then drag them to the easy-decal folder and drop them on the 'EasyDecal_dragNdrop.bat' file (the file will be highlighted when you are positioned correctly to drop the images).

Command Line

To create decals on any system using a command line:
IMPORTANT: Use Quotes in Windows
When running Easy Decal from a command prompt in Windows, any path and or file name that contains spaces must be surrounded by double quotes (the " character).
This means paths like C:\Program Files\easy-decal-@VERSION@ must be entered as "C:\Program Files\easy-decal-@VERSION@".
This is demonstrated in the example below.
1. Open a Command Prompt
On Windows this is done by going to Start -> Run, typing 'cmd' and pressing OK.
2. Ensure 'java' Is On The Path
This is done simply typing 'java -version' (without the quotes) and pressing enter.
If you are displayed several lines of version information, java is on the path.
If, instead, you receive the message "Bad command or file name", "'java' not found" or "'java' is not recognized as an internal or external command ...", java is NOT on the path, and may not even be installed.
If you don't know how to add java to the path, you should probably be using the drag 'n' drop method. ;)
3. Locate Your Source Images
On most systems, you can change to the directory containing your images using the 'cd' command.
4. Invoke Java to Run Easy Decal
Run Easy Decal over your images by executing the following Java command:
   java -showversion -jar <Easy Decal dir>\easy-decal.jar <image_file> [<image_file> ...]
where:

<Easy Decal dir>
is the directory where you installed Easy Decal (e.g. C:\Program Files\easy-decal-@VERSION@)

<image_file> [<image_file> ...]
is a list of images (at least one) from which you wish to create decals.
This can include wildcards (e.g. *.jpg).

Example:
   java -showversion -jar "C:\Program Files\easy-decal-@VERSION@\easy-decal.jar" *.jpg
If an OutOfMemoryError occus, try inserting the argument "-Xmx128m" after "-showversion".
